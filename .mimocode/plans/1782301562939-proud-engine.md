# MCP 工具调用机制分析

## 一、工具定义如何传递给大模型

### 流程图

```
mcpBuiltInTools (配置列表)
    ↓
mcpToolSettingsProvider (用户开关状态)
    ↓
ai_screen.dart._sendMessage() 构建 mcpTools 列表
    ↓
AiApiService.sendMessage() → 构建 HTTP 请求体
    ↓
请求体中包含 tools 参数，发送给大模型 API
```

### 具体代码路径

1. **`ai_screen.dart:1303-1340`** — 构建 tools 列表
   - 读取 `mcpEnabled`（总开关）和 `mcpToolConfigs`（各工具开关）
   - 遍历启用的工具，从 `mcpRegistry` 获取工具的 `name`、`description`、`inputSchema`
   - 构建为 OpenAI function calling 格式：
     ```json
     {"type": "function", "function": {"name": "xxx", "description": "xxx", "parameters": {...}}}
     ```

2. **`ai_api_service.dart:226-235`** — 放入请求体
   ```dart
   final requestData = {
     'model': settings.model,
     'messages': apiMessages,
     'tools': mcpTools,        // ← 工具定义放在这里
     'tool_choice': 'auto',    // ← 让模型自行决定是否调用工具
   };
   ```

## 二、大模型如何调用工具

大模型本身不执行代码，它只是在响应中返回**工具调用指令**，由应用端解析并执行。

### OpenAI Chat 协议（流式）

**`ai_api_service.dart:260-306`** — 流式解析中处理 tool_calls

```dart
// 从每个 SSE chunk 的 delta 中提取工具调用
final toolCalls = delta['tool_calls'] as List?;
if (toolCalls != null) {
  toolCallId = tc['id'];          // 工具调用 ID
  toolCallName = fn['name'];      // 工具名称
  toolCallArgsBuffer.write(fn['arguments']); // 参数（分块拼接）
}
```

流式结束后（`ai_api_service.dart:309-344`）：
1. 解析完整的工具调用参数
2. 调用 `onMcpToolCall(toolCallName, args)` 执行工具
3. 将结果作为 `role: 'tool'` 消息追加到上下文
4. **递归调用** `_openAIChatStream`，让模型基于工具结果生成最终回复

### OpenAI Chat 协议（非流式）

**`ai_api_service.dart:200-214`** — 直接处理 tool_calls

```dart
final toolCalls = message['tool_calls'] as List?;
for (final tc in toolCalls) {
  await _executeToolAndAppend(tc, apiMessages, onMcpToolCall);
}
```

### Anthropic 协议（流式）

**`ai_api_service.dart:670-800`** — Anthropic 的 tool_use 格式不同

```dart
// content_block_start 事件：识别工具调用开始
if (blockType == 'tool_use') {
  toolUseBuffers[id] = {'id': id, 'name': name, 'input': StringBuffer()};
}

// content_block_delta 事件：拼接工具参数
final partialJson = delta['partial_json'];

// message_stop 事件：stop_reason == 'tool_use' 时执行工具
```

## 三、工具执行的完整流程

```
用户发送消息
    ↓
构建请求体（含 tools 定义）→ 发送给大模型 API
    ↓
大模型返回 tool_calls（如 {"name": "now_time", "arguments": {...}}）
    ↓
应用端解析 tool_calls → 调用 onMcpToolCall 回调
    ↓
onMcpToolCall → mcpRegistry.get(name) → tool.execute(arguments)
    ↓
工具返回结果 → 追加为 role:'tool' 消息到上下文
    ↓
递归调用 API（上下文含工具结果）→ 大模型生成最终回复
    ↓
最终回复展示给用户
```

## 四、支持的 AI 协议

| 协议 | API 路径 | 工具调用格式 | 流式支持 |
|------|---------|-------------|---------|
| **OpenAI Chat** | `/chat/completions` | `tool_calls` (function calling) | ✅ SSE |
| **OpenAI Responses** | `/v1/responses` | `tool_calls` (function calling) | ❌ 仅非流式 |
| **Anthropic** | `/v1/messages` | `tool_use` (content_block) | ✅ SSE |

## 五、MCP Server Service（HTTP 服务端）

**`mcp_server_service.dart`** — 提供标准 MCP JSON-RPC 2.0 HTTP 服务

- **POST `/mcp`** — 处理 JSON-RPC 请求
  - `tools/list` — 返回所有已启用工具的定义
  - `tools/call` — 执行指定工具并返回结果
- **GET `/sse`** — SSE 端点（简化实现，回退到 POST 模式）

**注意**：当前项目的 MCP Server 是**本地 HTTP 服务**，供外部 MCP 客户端（如 Cursor、Claude Desktop）连接调用本应用的工具。而 AI 对话页面中的工具调用是**直接函数调用**（通过 `onMcpToolCall` 回调），不经过 HTTP。

## 六、两种工具调用路径总结

| 路径 | 使用场景 | 实现方式 |
|------|---------|---------|
| **AI 对话页面** | 用户与大模型聊天时 | `onMcpToolCall` 直接函数调用 → `tool.execute()` |
| **MCP HTTP 服务** | 外部 MCP 客户端连接时 | HTTP POST `/mcp` → JSON-RPC → `tool.execute()` |
