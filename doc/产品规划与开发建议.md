# 折多多 · 产品规划与开发建议

> 版本：v1.0 原型阶段 | 更新：2026-06-09 | 原型：`prototype/index.html`

---

## 一、产品定位

**折多多**是一款纯本地运行的商品折扣优惠记录工具，帮助用户集中管理各平台优惠信息，避免「收藏夹吃灰」和「错过好价」。

| 维度 | 说明 |
|------|------|
| 目标用户 | 精打细算的网购用户、数码/美妆爱好者、羊毛党 |
| 核心价值 | 本地记录 · 快速检索 · 离线可用 · 安全备份与云同步 |
| 差异化 | 无账号体系、无后端依赖、数据完全自主可控 |
| 平台 | iOS / Android / 鸿蒙（建议 Flutter 或 React Native 跨端） |

## 二、信息架构

```
折多多
├── 清单（Tab 0）
│   ├── 优惠列表（筛选 / 排序 / 快捷标签 / 双展示模式）

│   ├── 优惠详情（含视觉展示、促销权益、优惠券）
```

## 三、全面页面设计

### 3.1 清单首页（currentScreen = `list`，activeTab = 0）

**UI 布局：**
- **顶栏区**：左侧标题「优惠清单」+ 时间范围标签 + 记录数；右侧 3 个图标按钮（展示模式切换 / 搜索 / 筛选）
- **快捷筛选标签行**：横向可滚动的胶囊按钮 —— 全部 / 有优惠券 / 有图片 / 数码（类别快速过滤）
- **列表区**：支持两种展示模式
- **FAB 悬浮按钮**：右下角绿色圆形「+」按钮，始终显示

**双展示模式（设置项 `listDisplayMode`）：**

| 模式 | 每条内容 | 行高 | 适用场景 |
|------|----------|------|----------|
| `normal`（正常） | 标题 + 平台标签 + 标签(至多2) + 优惠券数 + 现价/原价/折扣 + 时间 + 编辑/删除按钮 | 较高 | 默认，信息完整 |
| `simple`（简单） | 标题(单行) + 原价 + 省金额 + 实付 | 紧凑 | 条目多时高效浏览 |

- **角标标注**：每张卡片右上角标注 `有图`(蓝色) / `ASCII`(紫色) / `无图`(灰色)
- normal 模式下卡片底部有: 创建日期 + 编辑/删除按钮；simple 模式无操作按钮

**交互逻辑：**
- 点击卡片 → 进入优惠详情
- 点击 FAB → 新建优惠
- 顶栏切换按钮可在两种展示模式间即时切换
- 顶栏搜索 → 进入搜索页（`goTo('search')`）
- 顶栏筛选 → 弹出底部 Sheet 筛选面板
- 有可用筛选条件时显示符合筛选的记录数

**状态覆盖：**
| 状态 | 显示 |
|------|------|
| 空数据 | 居中图标 + 「暂无优惠记录」+ 添加第一条按钮 |
| 筛选后无结果 | 「当前时间范围内暂无记录」+ 查看全部时间链接 |
| 加载中 | 无（纯前端过滤，无异步加载） |

### 3.2 搜索页（currentScreen = `search`）

**UI 布局：**
- **顶栏**：返回按钮 + 搜索输入框（带搜索图标） + 清除按钮（有输入时显示）
- **内容区**：未搜索时显示「最近搜索」+「热门标签」；搜索后显示结果列表

**交互逻辑：**
- 输入实时过滤（不触发网络请求），搜索范围覆盖：`title`, `platform`, `tags[]`, `category`, `coupons[*].source/strength/note`, `promotions[]`, `logistics`
- 点击最近搜索历史或热门标签 → 自动填入搜索框
- 搜索结果显示条数 + 匹配卡片列表
- 搜索结果卡片可点击进入详情
- 搜索历史存储在 localStorage（原型预设 3 条：'iPhone', '百亿补贴', '耳机'）
- 热门标签固定 5 个：百亿补贴、限时秒杀、数码、美妆、食品

**状态覆盖：**
| 状态 | 显示 |
|------|------|
| 未输入 | 最近搜索标签 + 热门标签区 |
| 有结果 | 结果数 + 结果列表 |
| 无结果 | 空列表提示（隐式通过结果列表为空） |

### 3.3 优惠详情（currentScreen = `detail`）

**UI 布局（从上到下）：**
- **顶栏**：返回 + 标题「优惠详情」+ 编辑/删除按钮
- **视觉区**（仅详情页展示）：
  - 图片模式（`visualType = 'image'`）：图片展示 + 压缩元信息（压缩前后体积）
  - ASCII 模式（`visualType = 'ascii'`）：代码块展示 + 标注「ASCII 图」
  - 无图模式：不显示视觉区
- **标题**：商品名（大号加粗）
- **标签行**：平台标签 + 自定义标签
- **价格区**：大号现价 + 原价划线 + 折扣标签（绿色）
- **节省金额**：原价 - 现价（仅当有原价时显示）
- **展示价**（如果有）：小字展示「展示价 ¥xxx」
- **促销权益**（如果有）：横向排列标签，灰色/绿底
- **优惠券**（如果有）：列表展示，每项含来源、张数、优惠力度、备注
- **信息列表**：
  - 物流（如果有）
  - 30 天销量（如果有）
  - 分类（始终显示）
  - 备注（如果有）
  - 购买链接（如果有）
- **复制链接按钮**：底部全宽绿色按钮（仅当有链接时显示）

**交互逻辑：**
- 点击编辑 → 进入编辑模式（预填当前数据）
- 点击删除 → 弹出删除确认对话框
- 点击复制链接 → Toast 提示「链接已复制」
- 所有价格字段支持 `originalPrice`、`currentPrice`、`currentDisplayPrice` 三种

**数据模型说明：**
```typescript
interface Coupon {
  count: number;           // 张数
  source: string;          // 来源：店铺券/平台券/直播间
  strength: string;        // 优惠力度：满300减50 / 9折
  note?: string;           // 券码、领取方式
}

interface Deal {
  id: string;
  title: string;
  platform: string;
  logistics?: string;
  category: string;
  originalPrice?: number;
  currentPrice: number;              // 到手价
  currentDisplayPrice?: number;      // 页面展示价
  currency?: string;
  discount: string;
  tags: string[];
  promotions?: string[];             // 原始促销文案
  link?: string;
  note?: string;
  coupons: Coupon[];
  sales?: { sold30Days: string };
  visualType: 'none' | 'image' | 'ascii';
  image?: string;                    // base64 data URL
  asciiArt?: string;
  imageMeta?: { originalSize: string; compressedSize: string; quality: number };
  createdAt: string;
  updatedAt: string;
}
```

### 3.4 新建/编辑优惠（currentScreen = `form`）

**新建与编辑的区别：**
- 新建时展示模式切换（YAML / 表单），编辑时仅显示表单（默认 manual 模式）
- 编辑时预填已有数据

#### 3.4.1 YAML 解析模式（新建推荐）

**UI 布局：**
- **提示条**：蓝色提示框，说明 YAML 支持范围
- **YAML 编辑区**：等宽字体文本框 + 「填入示例」按钮
- **解析按钮**：蓝色边框按钮「解析并填充」
- **解析状态**：错误提示（红色）或成功提示（绿色）
- **视觉内容区**（解析后自动填充）：模式选择（无/图片/ASCII）+ 图片上传或 URL 下载 + ASCII 编辑

**支持的 YAML 格式：**
- 结构化格式（含 `product` / `prices` / `promotions` / `source` / `visual` 层级）
- 简写扁平格式（直接 `title` / `currentPrice` / `platform` 等）

**YAML 解析逻辑（`yaml-parser.js`）：**

```yaml
# 结构化格式（推荐）
product:
  title: "商品名称"
  category: "家居"          # 可选，未填则自动推断
promotions:
  - "PLUS券后预计到手价1068元"
  - "领券再减15%"
  - "免费安装"
prices:
  original_price: 1629.00
  discounted_price: 1068.00
  current_display_price: 1492.25
  currency: "CNY"
source:
  platform: "京东"
  logistics: "京东物流"
  link: "https://..."
sales:
  sold_30_days: ">1000"
visual:
  type: image
  image_url: "https://..."
  ascii_art: |
    / \_/ \
   ( o.o )
     > ^ <
```

**Promotions 自动拆解逻辑（`parsePromotions`）：**
- 匹配 `免费安装|质保|保价|包邮|赠运|售后` 关键词 → 归类为 tags
- 匹配 `券|减|折|直降|到手价|满|赠送|PLUS|立减|秒杀` 关键词 → 归类为 coupons
- 每项 coupon 自动推断来源：`券/PLUS` → 优惠券；`直降/官方` → 官方直降；`满/赠送` → 满赠活动
- 分类自动推断（`inferCategory`）：通过标题关键词匹配家具/数码/家电/美妆/食品/服饰

**校验规则：**
- 商品名称必填
- 到手价必填
- YAML 必须可被 `js-yaml` 解析
- 解析成功后自动切换到表单模式

#### 3.4.2 表单填写模式（YAML 解析后 / 手动模式 / 编辑时）

**UI 布局（从上到下）：**
- 商品名称 *（输入框）
- 到手价 * / 原价（并列两列）
- 展示价 / 物流（并列两列，仅当有展示价时显示，编辑时始终显示）
- 平台 / 分类（下拉选择器）
- 购买链接
- 标签（逗号分隔输入框）
- 促销权益（多行文本框，每行一项）
- 优惠券（动态列表，可增删）
  - 每项：张数 + 来源 + 优惠力度 + 备注
- 视觉内容（模式选择 + 对应输入）
  - 图片：URL 输入 + 下载压缩按钮 / 本地文件选择 + Canvas 压缩展示
  - ASCII：多行文本编辑
- 备注（多行文本框）

**交互逻辑：**
- 顶栏：取消 / 标题（新建优惠/编辑优惠）/ 保存
- 保存时校验：名称和到手价必填
- 保存按钮行为：
  1. 如果当前是 YAML 模式且未解析 → 先执行 YAML 解析
  2. 如果 YAML 解析失败 → 阻止保存
  3. 构建数据 payload（自动计算折扣 `currentPrice / originalPrice * 10`）
  4. 编辑时更新现有 deal；新建时插入列表头部，自动生成 id(时间戳) 和 createdAt(当前日期)
- 优惠券可动态添加/删除

**图片处理逻辑（`image-utils.js`）：**
- 本地图片：FileReader → Canvas 调整最大宽 800px → JPEG 70% 质量 → base64 存储
- URL 图片：fetch(cors) → blob → FileReader → Canvas 压缩（同上流程）
- 压缩后保存原始尺寸、压缩尺寸、质量百分比
- **注意**：原型中跨域图片受 CORS 限制，正式 App 需使用原生 HTTP 库下载

### 3.5 AI 助手（currentScreen = `list`，activeTab = 1）

**交互架构：**
- 原型采用全屏 WebView **模拟**（非真实 iframe），地址栏 + 页面占位
- 正式 App 使用 WebView `loadUrl()` 全屏导航，禁止 iframe
- 原因：DeepSeek 等站点设置 `CSP frame-ancestors 'none'`，禁止 iframe 嵌入

**UI 布局（WebView 模拟）：**
- **浏览器工具栏**：刷新 + 后退(禁用) + 前进(禁用) + 地址栏(品牌色锁图标 + URL) + 浏览器打开
- **信息栏**：「WebView 全屏模式」+ AI 提供商名称
- **页面占位区**：
  - 加载动画（刷新时显示）
  - 品牌 logo + 提供商名 + 副标题「探索未至之境」
  - 登录表单模拟（手机号 + 验证码 + 登录按钮）
  - CSP 限制说明提示
- **底部操作栏**：
  - 主按钮：在浏览器打开（实际访问 AI 站点）
  - 副按钮：复制提示词 / AI 设置
  - 会话信息：上次访问时间 + 累计访问次数

**交互逻辑：**
- 切换至 AI Tab 时自动记录访问元数据（`persistAiSession`）
- 刷新按钮模拟加载状态（800ms 延迟后恢复）
- 浏览器打开使用 `window.open`，Cookie 由系统浏览器维护
- 点击「复制提示词」复制系统设置中的 `aiPromptPrefix` 到剪贴板
- 提示词预设为「图片商品信息解析引擎」，指导 AI 按结构化 YAML 输出

**会话持久化（`ai-session.js`）：**
- localStorage 存储 key `zdd_ai_session`
- 数据结构：`{ [providerId]: { providerId, embedUrl, lastVisited, visitCount, persistEnabled, mode, updatedAt } }`
- 支持导出/导入 JSON 备份文件（文件名 `zheduoduo_ai_session_YYYY-MM-DD.json`）
- 支持清除指定提供商的会话记录
- 正式 App：WebView 持久化分区保存 Cookie（Capacitor `partition: 'persist:zheduoduo-ai'`）

## 四、核心逻辑说明

### 4.1 数据持久化

| 数据 | localStorage Key | 说明 |
|------|------------------|------|
| 优惠记录 | `zdd_deals` | JSON 数组，含全部 Deal 字段 |
| 设置 | `zdd_settings` | 包含主题、AI 配置、云同步配置等 |
| AI 会话 | `zdd_ai_session` | 按提供商索引的会话元数据 |

- 所有数据纯前端存储，无后端依赖
- 使用 `watch` 深度监听 deals 变化自动保存
- 设置手动调用 `saveSettings()` 保存

### 4.2 筛选与排序（`filteredDeals`）

筛选顺序：快捷标签 → 平台 → 时间范围
排序支持 5 种模式。

### 4.3 搜索（`searchResults`）

全文检索，匹配规则：`title.includes(q) || platform.includes(q) || tags.includes(q) || category.includes(q) || coupons.some(...) || promotions.some(...) || logistics.includes(q)`

### 4.4 云同步架构

**增量同步（自动/手动触发）：**
- 基于 changelog 记录自上次同步以来的变更
- 仅传输变更数据 + 新增/修改的图片
- 原型模拟 Toast 提示，正式 App 实现 `sync_changelog` 表 + 文件差异

**全量同步（手动触发）：**
- 传输完整 SQLite 数据库 + 所有图片
- 覆盖目标端全部数据
- 操作前需二次确认弹窗

**同步方式对比：**

| 特性 | WebDAV | 腾讯云 COS | 阿里云 OSS |
|------|--------|------------|------------|
| 预设提供商 | 5 种 | 无 | 无 |
| 安全存储 | 系统密钥链 | 系统密钥链 | 系统密钥链 |
| 自动同步 | 支持 | 支持 | 支持 |
| 支持增量 | 是 | 是 | 是 |
| 复杂度 | 低（HTTP Basic Auth） | 中（SDK 签名） | 中（SDK 签名） |

### 4.5 AI 聊天图片处理逻辑

**触发方式：**
AI 聊天页面输入区左侧的「图片识别」按钮（原 OCR 按钮），点击后弹出底部 Sheet 选择处理方式。

**两种处理模式：**

| 模式 | 说明 | 适用场景 | 模型能力要求 |
|------|------|----------|--------------|
| OCR 识别 | 选择图片后异步 OCR，提取文字内容以文本形式带入会话 | 图片含文字信息（如商品截图、价格标签） | 无特殊要求，任何文本模型均可 |
| 多模态识别 | 选择图片后自动压缩，以 Base64 Data URI 形式随消息发送至 AI | 图片为视觉内容（如商品实物图、UI 截图） | 模型必须具备 `image` / `multimodal` / `vision` 能力之一 |

**多模态识别的技术实现：**
1. 图片经 `ImageUtils.prepareImage()` 按数据库压缩配置进行压缩（Windows/Linux 直接复制，移动端按质量/宽度压缩）
2. 压缩后的图片文件读取为字节数组，经 Base64 编码构建 Data URI：`data:image/jpeg;base64,{base64}`
3. Data URI 随最后一条用户消息的 `content` 数组以 `image_url` 类型传递（OpenAI Chat Completions / Responses 协议）或以 `image` + `source.base64` 传递（Anthropic 协议）
4. 单张图片估算 100 token，计入消息压缩预算

**能力检测与禁用逻辑：**
- 在 `_loadData()` 中从当前激活的 `ai_configs` 记录解析 `capabilities` JSON 数组
- 若数组包含 `image`、`multimodal`、`vision` 任一值，则「多模态识别」选项可用
- 否则该选项置灰，副标题提示「当前模型不支持图片/多模态能力」
- `capabilities` 字段在 AI 设置界面可手动编辑（标签式多选）

**待发送图片预览：**
- OCR 模式：缩略图 + 底部文字（识别中… / 无识别结果 / OCR 文本前 20 字）+ 右上角加载/完成指示器
- Base64 模式：缩略图 + 底部文字「多模态」+ 右下角黑色半透明「多模态」标签


### 5.3 布局规范

- 基准宽度：**375px**（iPhone 标准）
- 圆角：卡片 16px（`rounded-2xl`），按钮 12px（`rounded-xl`），标签全圆角
- 间距：页面边距 16px（`px-4`），卡片间距 12px
- 字号：标题 18-20px（`text-lg` ~ `text-xl`），正文 14px（`text-sm`），辅助 10-12px（`text-xs`）
- 底部 Tab 高度：49px + 34px Safe Area = 83px

### 5.4 交互原则

1. **单手操作友好**：主要操作在下半屏，FAB 右下角
2. **减少层级**：最多 2 层导航深度（Tab → 子页）
3. **即时反馈**：Toast 提示、按钮按压态（`active:scale-[.98]`）、列表空状态
4. **数据安全感**：删除/导入等破坏性操作必须二次确认
5. **过渡动画**：Sheet 从底部滑入（0.28s cubic-bezier）；Fade 遮罩（0.2s）
