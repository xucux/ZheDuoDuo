# 增量同步开发指南

## 一、概述

本文档为折多多（ZheDuoDuo）增量同步功能的**完整开发指南**，基于已冻结的设计方案。所有改动一次性完成，不拆分迭代。

**核心目标**：
1. 扩展现有 `sync_changelog` 表，覆盖所有业务表的变更记录。
2. 引入 `synclocal.json` / `synccloud.json` 版本协调机制。
3. 增量推送：本地未同步的 changelog + 附件打包为 ZIP 上传到云端 `changelog/`。
4. 增量拉取：拉取远程增量包并应用到本地数据库。
5. 全量同步上传后清空 changelog；全量下载前检测本地未同步变更并弹窗确认。
6. 乐观锁 + 重试解决多设备并发推送冲突。

## 二、数据库变更

### 2.1 `sync_changelog` 表扩展（Drift 定义）

在 `lib/core/database/tables/sync_changelog.dart` 中修改：

```dart
import 'package:drift/drift.dart';

class SyncChangelogs extends Table {
  IntColumn get id => integer().autoIncrement()();
  TextColumn get deviceId => text().named('device_id')();
  IntColumn get revision => integer()();

  // 新增字段
  TextColumn get tableName => text().named('table_name')();
  TextColumn get primaryKey => text().named('primary_key')(); // JSON 数组或字符串
  TextColumn get operation => text()(); // 'INSERT' / 'UPDATE' / 'DELETE'
  DateTimeColumn get changedAt => dateTime().named('changed_at')();
  DateTimeColumn get syncedAt => dateTime().named('synced_at').nullable()();
  IntColumn get hasAttachment => integer().named('has_attachment').withDefault(const Constant(0))();
  TextColumn get attachmentPaths => text().named('attachment_paths').nullable()(); // JSON 数组
  TextColumn get payload => text().nullable()(); // 变更数据快照（JSON）
}
```

**字段说明**：

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | INTEGER AUTOINCREMENT | 本地自增 ID |
| `device_id` | TEXT | 产生变更的设备 ID |
| `revision` | INTEGER | 设备本地单调递增版本号 |
| `table_name` | TEXT | 变更表名，如 `deals`、`app_settings` |
| `primary_key` | TEXT | 主键值。单字段主键直接存值；联合主键存 JSON 数组，如 `["dealId","sortOrder"]` |
| `operation` | TEXT | `INSERT` / `UPDATE` / `DELETE` |
| `changed_at` | DATETIME | 变更发生时间 |
| `synced_at` | DATETIME NULL | 推送成功时间，NULL 表示待同步 |
| `has_attachment` | INTEGER DEFAULT 0 | 是否涉及附件（1=是，0=否） |
| `attachment_paths` | TEXT NULL | 附件本地路径列表，JSON 数组格式 |
| `payload` | TEXT NULL | 变更后数据快照（JSON）。DELETE 时可为空 |

### 2.2 数据库迁移（schemaVersion 8 → 9）

在 `lib/core/database/app_database.dart` 的 `MigrationStrategy` 中增加：

```dart
onUpgrade: (m, from, to) async {
  if (from < 9) {
    // 新增字段
    await m.addColumn(syncChangelogs, syncChangelogs.tableName);
    await m.addColumn(syncChangelogs, syncChangelogs.primaryKey);
    await m.addColumn(syncChangelogs, syncChangelogs.operation);
    await m.addColumn(syncChangelogs, syncChangelogs.hasAttachment);
    await m.addColumn(syncChangelogs, syncChangelogs.attachmentPaths);
    await m.addColumn(syncChangelogs, syncChangelogs.payload);

    // 将旧数据中的 entity_type 映射到 table_name
    // 将旧数据中的 entity_id 映射到 primary_key
    await customStatement('''
      UPDATE sync_changelog
      SET table_name = entity_type,
          primary_key = json_array(entity_id),
          operation = CASE operation
            WHEN 'upsert' THEN 'UPDATE'
            WHEN 'pending_delete' THEN 'UPDATE'
            WHEN 'delete' THEN 'DELETE'
            ELSE 'UPDATE'
          END
      WHERE table_name IS NULL;
    ''');
  }
},
```

**注意**：
- `json_array` 在部分 SQLite 版本中不可用，若 Drift 的 SQLite 版本较低，可改为直接用 `entity_id` 字符串（单字段主键场景）。
- 旧数据中的 `entity_type` 只有 `deal`，迁移后统一映射为 `deals`。

## 三、核心组件设计

### 3.1 `ChangeLogger` 中间件

**职责**：拦截所有业务 DAO 的写操作，在云同步总开关开启时，向 `sync_changelog` 表写入变更记录。

**位置**：`lib/core/sync/change_logger.dart`

```dart
class ChangeLogger {
  final AppDatabase _db;
  final SyncMetaDao _syncMetaDao;
  final SettingsDao _settingsDao; // 用于读取云同步总开关

  ChangeLogger(this._db, this._syncMetaDao, this._settingsDao);

  /// 记录一次变更
  Future<void> log({
    required String tableName,
    required String primaryKey,
    required String operation, // INSERT / UPDATE / DELETE
    int? hasAttachment,
    List<String>? attachmentPaths,
    Map<String, dynamic>? payload,
  }) async {
    // 1. 检查云同步总开关
    final enabled = await _settingsDao.getCloudSyncEnabled();
    if (!enabled) return;

    // 2. 获取设备 ID
    final deviceId = await _syncMetaDao.getDeviceId();
    if (deviceId == null || deviceId.isEmpty) return;

    // 3. 获取下一个 revision
    final lastRevision = await _db.customSelect(
      'SELECT MAX(revision) as rev FROM sync_changelog WHERE device_id = ?',
      variables: [Variable(deviceId)],
    ).getSingleOrNull();
    final revision = (lastRevision?.data['rev'] as int?) ?? 0;

    // 4. 写入 changelog
    await _db.into(_db.syncChangelogs).insert(SyncChangelogsCompanion(
      deviceId: Value(deviceId),
      revision: Value(revision + 1),
      tableName: Value(tableName),
      primaryKey: Value(primaryKey),
      operation: Value(operation),
      changedAt: Value(DateTime.now()),
      syncedAt: const Value.absent(),
      hasAttachment: Value(hasAttachment ?? 0),
      attachmentPaths: Value(attachmentPaths != null ? jsonEncode(attachmentPaths) : null),
      payload: Value(payload != null ? jsonEncode(payload) : null),
    ));
  }
}
```

**集成方式**：在所有业务 DAO 的写操作方法末尾调用 `ChangeLogger.log()`。

**需要接入的 DAO 及调用点**：

| DAO | 写操作方法 | 调用示例 |
|-----|-----------|---------|
| `DealDao` | `saveDeal()` | `log(tableName: 'deals', primaryKey: dealId, operation: 'UPDATE', ...)` |
| `DealDao` | `softDeleteDeal()` | `log(tableName: 'deals', primaryKey: dealId, operation: 'UPDATE')` |
| `DealDao` | `hardDeleteDeal()` | `log(tableName: 'deals', primaryKey: dealId, operation: 'DELETE')` |
| `DealDao` | `restoreDeal()` | `log(tableName: 'deals', primaryKey: dealId, operation: 'UPDATE')` |
| `DealDao` | `saveDealImages()` | `log(tableName: 'deal_images', primaryKey: dealId, operation: 'UPDATE', hasAttachment: 1, attachmentPaths: [...])` |
| `SettingsDao` | `setValue()` | `log(tableName: 'app_settings', primaryKey: key, operation: 'UPDATE', payload: {'value': value})` |
| `AiConfigDao` | `upsert()` | `log(tableName: 'ai_configs', primaryKey: id, operation: 'UPDATE', payload: {...})` |
| `SecretsDao` | `upsert()` | `log(tableName: 'secrets', primaryKey: id.toString(), operation: 'UPDATE', payload: {...})` |
| `PromptDao` | `upsert()` | `log(tableName: 'prompts', primaryKey: id, operation: 'UPDATE', payload: {...})` |
| `ImageCompressDao` | `upsert()` | `log(tableName: 'image_compress_settings', primaryKey: minSize.toString(), operation: 'UPDATE', payload: {...})` |
| `BackupDao` | `insert()` | `log(tableName: 'backup_records', primaryKey: id.toString(), operation: 'INSERT', payload: {...})` |

**注意**：
- `deal_tags`、`deal_promotions`、`coupons` 的变更通常随 `deals` 的 `saveDeal()` 一起发生。可在 `saveDeal()` 的事务结束后，统一记录 `deals` 主表的变更，附件标记以 `deal_images` 为准。
- `backup_records` 是否需要同步视业务需求而定，若仅本地备份记录无需同步，可跳过。

### 3.2 `SyncStateManager`（状态文件管理）

**职责**：管理本地 `synclocal.json` 和远程 `synccloud.json` 的读取、更新、初始化。

**位置**：`lib/core/sync/sync_state_manager.dart`

```dart
class SyncStateManager {
  final CloudStorageService _cloudStorage; // 现有云存储服务
  final String _deviceId;

  SyncStateManager(this._cloudStorage, this._deviceId);

  /// 本地状态文件路径
  String get _localStatePath => '...'; // app 文档目录下的 synclocal.json

  /// 远程状态文件路径
  String get _remoteStatePath => 'synccloud.json';

  /// 读取本地状态
  Future<SyncState> readLocalState() async {
    final file = File(_localStatePath);
    if (!await file.exists()) return SyncState.empty(_deviceId);
    final json = jsonDecode(await file.readAsString());
    return SyncState.fromJson(json);
  }

  /// 写入本地状态
  Future<void> writeLocalState(SyncState state) async {
    final file = File(_localStatePath);
    await file.writeAsString(jsonEncode(state.toJson()));
  }

  /// 拉取远程状态
  Future<SyncState?> fetchRemoteState() async {
    try {
      final content = await _cloudStorage.readFile(_remoteStatePath);
      if (content == null || content.isEmpty) return null;
      return SyncState.fromJson(jsonDecode(content));
    } catch (_) {
      return null;
    }
  }

  /// 写入远程状态（乐观锁：基于预期 version 写回）
  Future<bool> writeRemoteState(SyncState newState, {required int expectedVersion}) async {
    try {
      // 先读取远程当前状态
      final remote = await fetchRemoteState();
      if (remote != null && remote.version != expectedVersion) {
        return false; // 冲突，需要重试
      }
      await _cloudStorage.writeFile(_remoteStatePath, jsonEncode(newState.toJson()));
      return true;
    } catch (_) {
      return false;
    }
  }

  /// 初始化本地状态（首次全量同步后调用）
  Future<SyncState> initializeLocalState() async {
    final state = SyncState.empty(_deviceId);
    await writeLocalState(state);
    return state;
  }
}
```

### 3.3 `IncrementalSyncService`（增量同步服务）

**职责**：增量推送、增量拉取、乐观锁重试、附件打包/解包。

**位置**：`lib/core/sync/incremental_sync_service.dart`

```dart
class IncrementalSyncService {
  final AppDatabase _db;
  final SyncStateManager _stateManager;
  final CloudStorageService _cloudStorage;
  final String _deviceId;

  IncrementalSyncService(this._db, this._stateManager, this._cloudStorage, this._deviceId);

  /// ========== 增量推送（核心入口）==========
  Future<SyncResult> pushIncremental() async {
    // 1. 读取本地状态
    var localState = await _stateManager.readLocalState();

    // 2. 读取待同步的 changelog
    final pendingChanges = await _getPendingChangelogs();
    if (pendingChanges.isEmpty) {
      return SyncResult.noChanges();
    }

    // 3. 打包增量数据
    final zipPath = await _packIncrementalZip(pendingChanges);
    if (zipPath == null) {
      return SyncResult.failure('打包增量包失败');
    }

    // 4. 上传 ZIP
    final zipFileName = p.basename(zipPath);
    final remoteZipPath = 'changelog/$zipFileName';
    final uploadSuccess = await _cloudStorage.uploadFile(zipPath, remoteZipPath);
    if (!uploadSuccess) {
      return SyncResult.failure('上传增量包失败');
    }

    // 5. 乐观锁更新 synccloud.json
    final expectedVersion = localState.version;
    final newRemoteState = _buildNewRemoteState(
      baseState: localState,
      newZipFile: zipFileName,
    );

    final writeSuccess = await _stateManager.writeRemoteState(
      newRemoteState,
      expectedVersion: expectedVersion,
    );

    if (!writeSuccess) {
      // 乐观锁冲突：删除刚上传的孤儿 ZIP，然后重试
      await _cloudStorage.deleteFile(remoteZipPath);
      return SyncResult.conflict();
    }

    // 6. 标记本地 changelog 为已同步
    await _markChangelogsSynced(pendingChanges);

    // 7. 更新本地 synclocal.json
    localState = newRemoteState;
    await _stateManager.writeLocalState(localState);

    return SyncResult.success();
  }

  /// ========== 增量拉取（核心入口）==========
  Future<SyncResult> pullIncremental() async {
    // 1. 读取本地和远程状态
    final localState = await _stateManager.readLocalState();
    final remoteState = await _stateManager.fetchRemoteState();

    if (remoteState == null) {
      return SyncResult.noChanges();
    }

    // 2. 比较版本
    if (remoteState.version <= localState.version) {
      return SyncResult.noChanges();
    }

    // 3. 计算需要拉取的增量包
    final remoteZips = remoteState.changelogZip;
    final localZips = localState.changelogZip;
    final pendingZips = remoteZips.where((z) => !localZips.contains(z)).toList();

    if (pendingZips.isEmpty) {
      return SyncResult.noChanges();
    }

    // 4. 逐个下载并应用
    for (final zipName in pendingZips) {
      final success = await _downloadAndApplyZip('changelog/$zipName');
      if (!success) {
        return SyncResult.failure('拉取增量包失败: $zipName');
      }
    }

    // 5. 更新本地 synclocal.json（同步为远程版本）
    final updatedLocalState = localState.copyWith(
      version: remoteState.version,
      lastSyncDeviceId: remoteState.lastSyncDeviceId,
      lastSyncTime: DateTime.now(),
      changelogZip: List.from(remoteState.changelogZip),
      deviceSyncLog: _mergeDeviceSyncLog(localState.deviceSyncLog, remoteState.deviceSyncLog),
    );
    await _stateManager.writeLocalState(updatedLocalState);

    return SyncResult.success();
  }

  /// ========== 智能同步（拉取 + 推送）==========
  Future<SyncResult> smartSync() async {
    // 1. 先拉取
    final pullResult = await pullIncremental();
    if (pullResult.isFailure) return pullResult;

    // 2. 再推送（本地可能有未同步的变更）
    final pushResult = await pushIncremental();
    return pushResult;
  }
}
```

## 五、ZIP 打包与解包

### 5.1 打包逻辑

```dart
Future<String?> _packIncrementalZip(List<SyncChangelogData> changes) async {
  final tmpDir = await getTemporaryDirectory();
  final timestamp = DateTime.now().millisecondsSinceEpoch;
  final zipName = 'changelog_${_deviceId}_${localVersion + 1}_$timestamp.zip';
  final zipPath = p.join(tmpDir.path, zipName);

  final encoder = ZipFileEncoder();
  encoder.create(zipPath);

  // 1. 写入 JSONL 数据文件
  final jsonlPath = p.join(tmpDir.path, 'changes.jsonl');
  final jsonlFile = File(jsonlPath);
  final buffer = StringBuffer();
  final attachmentSet = <String>{}; // 去重

  for (final change in changes) {
    buffer.writeln(jsonEncode({
      'device_id': change.deviceId,
      'revision': change.revision,
      'table_name': change.tableName,
      'primary_key': change.primaryKey,
      'operation': change.operation,
      'changed_at': change.changedAt.toIso8601String(),
      'has_attachment': change.hasAttachment,
      'attachment_paths': change.attachmentPaths,
      'payload': change.payload,
    }));

    if (change.hasAttachment == 1 && change.attachmentPaths != null) {
      final paths = jsonDecode(change.attachmentPaths!) as List;
      for (final path in paths) {
        attachmentSet.add(path as String);
      }
    }
  }

  await jsonlFile.writeAsString(buffer.toString());
  encoder.addFile(jsonlFile);

  // 2. 写入附件（平铺到 ZIP 根目录，去重）
  for (final path in attachmentSet) {
    final file = File(path);
    if (await file.exists()) {
      final fileName = p.basename(file.path);
      encoder.addFile(file, fileName);
    }
  }

  encoder.close();

  // 清理临时 JSONL 文件
  await jsonlFile.delete();

  return zipPath;
}
```

### 5.2 解包逻辑

```dart
Future<bool> _downloadAndApplyZip(String remoteZipPath) async {
  try {
    final tmpDir = await getTemporaryDirectory();
    final zipName = p.basename(remoteZipPath);
    final localZipPath = p.join(tmpDir.path, zipName);

    // 1. 下载 ZIP
    await _cloudStorage.downloadFile(remoteZipPath, localZipPath);

    // 2. 解压
    final extractDir = p.join(tmpDir.path, 'extract_$zipName');
    await Directory(extractDir).create(recursive: true);
    await ZipFile.extractToDirectory(
      zipFile: File(localZipPath),
      destinationDir: Directory(extractDir),
    );

    // 3. 读取 JSONL 并应用
    final jsonlFile = File(p.join(extractDir, 'changes.jsonl'));
    if (!await jsonlFile.exists()) return false;

    final lines = await jsonlFile.readAsLines();
    for (final line in lines) {
      if (line.trim().isEmpty) continue;
      final change = jsonDecode(line);
      await _applyChange(change, extractDir);
    }

    // 4. 清理临时文件
    await File(localZipPath).delete();
    await Directory(extractDir).delete(recursive: true);

    return true;
  } catch (e) {
    return false;
  }
}

Future<void> _applyChange(Map<String, dynamic> change, String extractDir) async {
  final tableName = change['table_name'] as String;
  final operation = change['operation'] as String;
  final primaryKey = change['primary_key'] as String;
  final payload = change['payload'] != null
    ? jsonDecode(change['payload'] as String) as Map<String, dynamic>
    : null;

  switch (tableName) {
    case 'deals':
      await _applyDealChange(operation, primaryKey, payload);
      break;
    case 'app_settings':
      await _applySettingsChange(operation, primaryKey, payload);
      break;
    // ... 其他表
  }

  // 附件处理：将 ZIP 中的附件复制到本地图片目录
  if (change['has_attachment'] == 1 && change['attachment_paths'] != null) {
    final paths = jsonDecode(change['attachment_paths'] as String) as List;
    for (final originalPath in paths) {
      final fileName = p.basename(originalPath as String);
      final extractedFile = File(p.join(extractDir, fileName));
      if (await extractedFile.exists()) {
        final localDir = await getApplicationDocumentsDirectory();
        final targetPath = p.join(localDir.path, 'images', fileName);
        await Directory(p.dirname(targetPath)).create(recursive: true);
        await extractedFile.copy(targetPath);
      }
    }
  }
}
```

## 六、冲突解决

### 6.1 策略：Last Write Wins（LWW）

当本地和远程修改了同一条记录时：

1. 拉取远程变更，按 `changed_at` 时间戳比较。
2. 远程 `changed_at` 更新 → 用远程 payload 覆盖本地。
3. 本地 `changed_at` 更新 → 保留本地，待推送时覆盖远程。
4. `changed_at` 完全相同时 → 按 `deviceId` 字典序决定（确定性裁决）。

### 6.2 应用远程变更时的冲突检测

```dart
Future<void> _applyDealChange(String operation, String primaryKey, Map<String, dynamic>? payload) async {
  if (operation == 'DELETE') {
    await _db.into(_db.deals).deleteWhere((t) => t.id.equals(primaryKey));
    return;
  }

  if (payload == null) return;

  // 查询本地是否存在
  final localDeal = await (_db.select(_db.deals)..where((t) => t.id.equals(primaryKey))).getSingleOrNull();

  if (localDeal == null) {
    // 本地不存在，直接插入
    await _db.into(_db.deals).insert(DealsCompanion(
      id: Value(primaryKey),
      // ... 从 payload 映射其他字段
    ), mode: InsertMode.insertOrReplace);
  } else {
    // 本地存在，比较时间戳
    final remoteUpdatedAt = DateTime.tryParse(payload['updated_at'] as String? ?? '');
    if (remoteUpdatedAt != null && localDeal.updatedAt != null && remoteUpdatedAt.isAfter(localDeal.updatedAt!)) {
      // 远程更新，覆盖本地
      await _db.into(_db.deals).insert(DealsCompanion(
        id: Value(primaryKey),
        // ... 从 payload 映射
      ), mode: InsertMode.insertOrReplace);
    }
    // 否则本地更新，不做任何操作，等待下次推送
  }
}
```

## 七、错误处理与边界情况

### 7.1 乐观锁冲突（Concurrent Push）

当 `pushIncremental()` 返回 `SyncResult.conflict()` 时，外层调用者应：

```dart
Future<SyncResult> pushWithRetry({int maxRetries = 3}) async {
  for (var i = 0; i < maxRetries; i++) {
    final result = await pushIncremental();
    if (!result.isConflict) return result;

    // 冲突后先拉取远程最新状态
    await pullIncremental();
  }
  return SyncResult.failure('推送失败，并发冲突超过最大重试次数');
}
```

### 7.2 网络中断（原子性）

| 中断时机 | 影响 | 恢复策略 |
|---------|------|---------|
| ZIP 上传前 | 无影响 | 直接重试 |
| ZIP 上传成功，但 synccloud.json 未更新 | 产生孤儿 ZIP | 下次推送时，清理 changelogZip 中不存在的孤儿文件 |
| synccloud.json 更新成功，但本地 changelog 未标记 | 下次推送会重复上传 | 幂等：同一 revision 的 ZIP 覆盖上传即可 |
| 拉取 ZIP 后应用中断 | 部分变更已应用 | 基于 `revision` + `deviceId` 去重，已应用的跳过 |

### 7.3 孤儿 ZIP 清理

每次推送前，扫描远程 `changelog/` 目录中的 ZIP 文件。若某 ZIP 不在 `synccloud.json` 的 `changelogZip` 列表中，且超过 7 天，则删除。

### 7.4 长时间未同步设备

若设备超过 30 天未同步，期间的增量包可能已被清理。此时：
1. 弹窗提示用户："增量记录已过期，建议先进行全量同步"。
2. 引导用户执行全量下载，以全量备份作为新基线。
